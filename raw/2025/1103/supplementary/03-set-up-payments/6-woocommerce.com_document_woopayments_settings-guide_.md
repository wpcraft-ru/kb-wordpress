# WooPayments: Settings Guide

**[Spanish version of this page](https://woocommerce.com/document/woopayments/woopayments-guia-de-configuracion/) |[Aquí puedes encontrar la versión en español de esta página](https://woocommerce.com/document/woopayments/woopayments-guia-de-configuracion/)**

WooPayments is designed to be fairly straightforward and intuitive to use. However, there are still some settings that each merchant will want to configure to their liking. This page is intended to explain in detail what each option does and how it might be used.

The WooPayments settings can be found under **Payments > Settings**.

**NOTE:** After adjusting any WooPayments settings, you will need to scroll to the bottom of the page and click the **Save Changes** button. Any changes to the settings will not take effect until they are saved!

## [General](https://woocommerce.com/document/woopayments/settings-guide/#general)

[↑ Back to top](https://woocommerce.com/document/woopayments/settings-guide/#doc-title "Back to top")
This section contains some basic options that likely won’t need to be changed very often.

[![Image 13](https://woocommerce.com/wp-content/uploads/2025/03/Screenshot-taken-on-2025-04-12-at-13.32.20-UTC@2x.png?strip=all&w=704)](https://woocommerce.com/wp-content/uploads/2025/03/Screenshot-taken-on-2025-04-12-at-13.32.20-UTC@2x.png)
### [Enable WooPayments](https://woocommerce.com/document/woopayments/settings-guide/#enable-woopayments)

[↑ Back to top](https://woocommerce.com/document/woopayments/settings-guide/#doc-title "Back to top")
Unchecking this option will make it so that WooPayments is no longer offered as a payment method during checkout. This means that, if the box is unchecked, customers will not be able to pay via WooPayments.

What this setting does **not** do is disable the WooPayments plugin entirely. In other words, even with this box unchecked, you will still be able to access the **Payouts**, **Transactions** and **Disputes** pages, as well as issue refunds and so on.

If you want to completely disable the WooPayments plugin, please see [this page](https://woocommerce.com/document/woopayments/account-management/delete-account/).

### [Test mode](https://woocommerce.com/document/woopayments/settings-guide/#test-mode)

[↑ Back to top](https://woocommerce.com/document/woopayments/settings-guide/#doc-title "Back to top")
Check this box to enable test mode. Test mode lets you try out payments, refunds, disputes and other such processes as you’re working on your store without handling live payment information. Unchecking this box disables test mode and switches your site into live mode, meaning that it will accept real payment methods and actually charge your customers.

Please see our [Testing WooPayments](https://woocommerce.com/document/woopayments/testing-and-troubleshooting/testing/) page for details on how to use test mode.

## [Payments accepted on checkout](https://woocommerce.com/document/woopayments/settings-guide/#payments-methods)

[↑ Back to top](https://woocommerce.com/document/woopayments/settings-guide/#doc-title "Back to top")
This section allows you to add and edit the payment methods that customers can use to complete purchases on your site.

[![Image 14](https://woocommerce.com/wp-content/uploads/2024/08/Screenshot-taken-on-2026-03-11-at-17.45.36-UTC@2x.png?strip=all&w=704)](https://woocommerce.com/wp-content/uploads/2024/08/Screenshot-taken-on-2026-03-11-at-17.45.36-UTC@2x.png)
By checking the boxes in this section, you can also offer [additional payment methods](https://woocommerce.com/document/woopayments/payment-methods/additional-payment-methods/) to certain customers. Offering these payment options can increase the chances that visitors to your site will purchase from your store.

**NOTE:** It is not currently possible to uncheck the **Credit card / debit card** option.

## [Buy now, pay later](https://woocommerce.com/document/woopayments/settings-guide/#bnpl)

[↑ Back to top](https://woocommerce.com/document/woopayments/settings-guide/#doc-title "Back to top")
This section contains buy now, pay later payment (BNPL) methods. These can help boost sales by offering customers more buying power and flexible payment options.

[![Image 15](https://woocommerce.com/wp-content/uploads/2025/03/Screenshot-taken-on-2025-04-12-at-13.33.51-UTC@2x.png?strip=all&w=704)](https://woocommerce.com/wp-content/uploads/2025/03/Screenshot-taken-on-2025-04-12-at-13.33.51-UTC@2x.png)
Please see [our BNPL documentation](https://woocommerce.com/document/woopayments/payment-methods/buy-now-pay-later/) to learn more about these payment methods.

## [Express checkouts](https://woocommerce.com/document/woopayments/settings-guide/#express-checkouts)

[↑ Back to top](https://woocommerce.com/document/woopayments/settings-guide/#doc-title "Back to top")
This section controls which express checkouts are used on your site and how they look.

[![Image 16](https://woocommerce.com/wp-content/uploads/2024/08/Screenshot-taken-on-2026-04-21-at-15.33.10-UTC@2x.png?strip=all&w=704)](https://woocommerce.com/wp-content/uploads/2024/08/Screenshot-taken-on-2026-04-21-at-15.33.10-UTC@2x.png)
Express checkout methods are so named because they allow customers to check out faster than if the customer had not used one of these methods. WooPayments has the following express checkout methods built in:

*   [Amazon Pay](https://woocommerce.com/document/woopayments/payment-methods/amazon-pay/)
*   [Apple Pay](https://woocommerce.com/document/woopayments/payment-methods/apple-pay/)
*   [Google Pay](https://woocommerce.com/document/woopayments/payment-methods/google-pay/)
*   [Link by Stripe](https://woocommerce.com/document/woopayments/payment-methods/link-by-stripe/)
*   [WooPay](https://woocommerce.com/document/woopay-merchant-documentation/)

Please see the links above for more info on each method and its settings.

**NOTE:** WooPay and Link by Stripe are currently incompatible. You cannot have both express checkout methods enabled at the same time.

## [Transactions](https://woocommerce.com/document/woopayments/settings-guide/#transactions)

[↑ Back to top](https://woocommerce.com/document/woopayments/settings-guide/#doc-title "Back to top")
This section contains a settings related to how transactions function on your site, and how they appear to customers in emails, receipts, and bank statements.

[![Image 17](https://woocommerce.com/wp-content/uploads/2025/03/Screenshot-taken-on-2025-04-12-at-13.35.22-UTC@2x.png?strip=all&w=704)](https://woocommerce.com/wp-content/uploads/2025/03/Screenshot-taken-on-2025-04-12-at-13.35.22-UTC@2x.png)
### [Transaction preferences](https://woocommerce.com/document/woopayments/settings-guide/#transaction-preferences)

[↑ Back to top](https://woocommerce.com/document/woopayments/settings-guide/#doc-title "Back to top")
**Enable payments via saved cards**, if checked, allows customers to save their card info to their account on your site, which allows them to check out much faster if they make more purchases from you in the future.

**NOTE:** As detailed in our [PCI compliance doc](https://woocommerce.com/document/woopayments/our-policies/pci-compliance/#what-about-saved-cards-and-subscriptions), the actual credit card number and other details are stored on our secure server. They are presented to customers on your site via a secure tokenization method.

The **Enable manual capture** setting, if checked, prevents WooPayments from automatically capturing charges. This allows you to manually capture the charge at a later time, such as when you ship the item.

If you enable this, it’s important to know that **you will need to capture each order within 7 days of it being placed**. Otherwise, the authorization and the order itself will be cancelled.

**NOTE:** We have [a detailed guide to this workflow](https://woocommerce.com/document/woopayments/settings-guide/authorize-and-capture/) which you should review before enabling this setting. Our suggestion is that you **leave this option disabled** unless you have a very specific and well-justified use case in mind.

### [Customer statements](https://woocommerce.com/document/woopayments/settings-guide/#customer-statements)

[↑ Back to top](https://woocommerce.com/document/woopayments/settings-guide/#doc-title "Back to top")
The **Customer bank statement** setting controls how charges from your store appear on bank statements or in online banking systems.

By default, WooPayments will use the site URL for the statement descriptor. In the event that no statement descriptor is available, `WooPayments` will be used. You can learn more about the default `WooPayments` statement descriptor [here](https://woocommerce.com/document/woopayments/fees-and-debits/woopayid-on-statement/).

You can change the statement descriptor if need be, but it must adhere to these rules:

*   Contain only[Latin characters](https://en.wikipedia.org/wiki/Latin_alphabet). 
    *   Japanese merchants will also see descriptor fields for kanji and kana.

*   Contain between 5 and 22 characters, inclusive.
*   Contain at least one letter.
*   Not contain any of these characters:`< > \ ' " *`
*   Reflect your business name clearly.

Please see [this page](https://woocommerce.com/document/woopayments/customization-and-translation/bank-statement-descriptor/) for more details on recommended statement descriptors.

The **Support email** and **Support phone number** settings determine what contact information is shown on receipts, invoices, and automated emails from your store.

We strongly suggest filling these settings out, since they can help prevent disputes from customers who might otherwise have trouble contacting you for assistance. If you don’t have a dedicated support email or phone number, we still recommend adding _something_ here, even if it’s a personal email or phone number.

**NOTE:** The customer support email and phone number settings covered here are not the same as updating the email or phone number that you have on file in your Stripe Express account. To do that, please see [this page](https://woocommerce.com/document/woopayments/account-management/update-my-contact-details/).

## [Payouts](https://woocommerce.com/document/woopayments/settings-guide/#payouts)

[↑ Back to top](https://woocommerce.com/document/woopayments/settings-guide/#doc-title "Back to top")
The _Payouts_ section controls how often WooPayments pays out money from incoming orders into your linked bank account or debit card.

[![Image 18](https://woocommerce.com/wp-content/uploads/2025/03/Screenshot-taken-on-2025-04-12-at-13.35.57-UTC@2x.png?strip=all&w=704)](https://woocommerce.com/wp-content/uploads/2025/03/Screenshot-taken-on-2025-04-12-at-13.35.57-UTC@2x.png)
### [Payout schedule](https://woocommerce.com/document/woopayments/settings-guide/#payout-schedule)

[↑ Back to top](https://woocommerce.com/document/woopayments/settings-guide/#doc-title "Back to top")
This setting allows you to specify a payout schedule that works best for your business. You can find more information on the options in our dedicated [payout schedule document](https://woocommerce.com/document/woopayments/payouts/change-payout-schedule/).

If it’s been less than 7 days since your first transaction with WooPayments, you will not be able to change your payout schedule until [the initial 7 day waiting period](https://woocommerce.com/document/woopayments/payouts/payout-schedule/#new-accounts) has passed.

### [Payout bank account](https://woocommerce.com/document/woopayments/settings-guide/#payout-bank-account)

[↑ Back to top](https://woocommerce.com/document/woopayments/settings-guide/#doc-title "Back to top")
Your bank account or debit card details are securely stored and managed with our payments partner, Stripe. You can use the link in this section to [access the Stripe dashboard](https://woocommerce.com/document/woopayments/account-management/access-stripe-account/), through which you can view and update your bank account or debit card information, if need be.

## [Account notifications](https://woocommerce.com/document/woopayments/settings-guide/#account-notifications)

[↑ Back to top](https://woocommerce.com/document/woopayments/settings-guide/#doc-title "Back to top")
WooPayments sends various notifications to you via email, e.g. about disputes and so on. The _Account notifications_ setting controls which email address we use for those.

[![Image 19](https://woocommerce.com/wp-content/uploads/2024/08/Screenshot-taken-on-2026-03-11-at-14.30.35-UTC@2x.png?strip=all&w=704)](https://woocommerce.com/wp-content/uploads/2024/08/Screenshot-taken-on-2026-03-11-at-14.30.35-UTC@2x.png)

**NOTE:** The _Account notifications_ email is treated as an owner-level contact for support purposes. Anyone with access to this address can request account changes or retrieve account information through our support team, so it should only be set to an address controlled by an authorized account owner.

If you want to change which email address we send WooPayments-related notifications to, simply enter a new email in this field, enter the address again in the second field to confirm, and click **Save Changes** at the bottom. The old email address will be sent one last notification informing them of the change.

Note that this setting only controls where WooPayments emails go. Other WordPress or WooCommerce notification emails are not affected. Also, changing this setting is _not_ the same as [changing the personal or account email](https://woocommerce.com/document/woopayments/account-management/update-my-contact-details/) for your Stripe Express account.

## [Fraud protection](https://woocommerce.com/document/woopayments/settings-guide/#fraud-protection)

[↑ Back to top](https://woocommerce.com/document/woopayments/settings-guide/#doc-title "Back to top")
Our _Fraud protection_ feature offers rules which you can use to block suspicious orders before they get placed. This can help you avoid[carding attacks](https://woocommerce.com/document/woopayments/fraud-and-disputes/card-testing/)and increased disputes.

[![Image 20](https://woocommerce.com/wp-content/uploads/2025/03/Screenshot-taken-on-2025-04-12-at-13.36.28-UTC@2x.png?strip=all&w=704)](https://woocommerce.com/wp-content/uploads/2025/03/Screenshot-taken-on-2025-04-12-at-13.36.28-UTC@2x.png)
In short, when an order is placed, the fraud protection engine will evaluate that order according to the risk level you’ve selected here. Depending on your configuration, the order can either be allowed as normal or blocked outright.

We suggest that most sites use the**Basic**risk level, but you can optionally use the **Advanced** setting to configure specific fraud protection rules to best suit your business type, risk tolerance, and other factors. To learn more, please consult[our dedicated page](https://woocommerce.com/document/woopayments/fraud-and-disputes/fraud-protection/).

## [Advanced settings](https://woocommerce.com/document/woopayments/settings-guide/#advanced-settings)

[↑ Back to top](https://woocommerce.com/document/woopayments/settings-guide/#doc-title "Back to top")
The _Advanced settings_ section gives you further flexibility in managing your WooPayments configuration. While you may not interact with these settings regularly, you can find them at the bottom of the settings page.

[![Image 21](https://woocommerce.com/wp-content/uploads/2024/08/Screenshot-taken-on-2025-04-12-at-14.03.08-UTC@2x.png?strip=all&w=704)](https://woocommerce.com/wp-content/uploads/2024/08/Screenshot-taken-on-2025-04-12-at-14.03.08-UTC@2x.png)
### [Enable multi-currency](https://woocommerce.com/document/woopayments/settings-guide/#enable-multicurrency)

[↑ Back to top](https://woocommerce.com/document/woopayments/settings-guide/#doc-title "Back to top")
Enabling this lets you to accept payments from customers in currencies besides your [shop currency](https://woocommerce.com/document/shop-currency/). Please see our docs on [accepting multiple currencies](https://woocommerce.com/document/woopayments/currencies/#accepting-multiple-currencies) for more information.

### [Enable Stripe Billing for future subscriptions](https://woocommerce.com/document/woopayments/settings-guide/#stripe-billing)

[↑ Back to top](https://woocommerce.com/document/woopayments/settings-guide/#doc-title "Back to top")
This setting is only shown if you have [Woo Subscriptions](https://woocommerce.com/products/woocommerce-subscriptions/) activated in addition to WooPayments. If you have both, you can choose between the on-site billing engine provided by Woo Subscriptions, or the off-site engine powered by Stripe Billing.

Please see [our Stripe Billing document](https://woocommerce.com/document/woopayments/subscriptions/stripe-billing/) for full details on this feature.

### [Debug mode / Log error messages](https://woocommerce.com/document/woopayments/settings-guide/#debug-mode)

[↑ Back to top](https://woocommerce.com/document/woopayments/settings-guide/#doc-title "Back to top")
With this option enabled, WooPayments will write debugging information to a log file. During normal operations, this is not necessary, but our support staff may direct you to enable debug mode if you contact us about an issue with WooPayments.

To view the log files and send them to us:

1.   Go to **WooCommerce > Status > Logs** in your site’s dashboard.
2.   Find the WooPayments log from the relevant date and click to open it.
3.   Click **Download** to save the log file to your computer.
4.   Attach that file to your email response to our support staff.

After the relevant issue has been resolved, feel free to disable debug mode again.

**NOTE:**By default, log files are only kept for 30 days.

## [Address autocomplete](https://woocommerce.com/document/woopayments/settings-guide/#address-autocompletion)

[↑ Back to top](https://woocommerce.com/document/woopayments/settings-guide/#doc-title "Back to top")
WooPayments also enables one setting outside of the **Payments > Settings** page, which is an address autocomplete option for the checkout page.

This streamlines the checkout experience via predictive address search, which reduces customer friction and as well as improves address accuracy. It requires WooCommerce 10.3 or later, and WooPayments must be[enabled in checkout](https://woocommerce.com/document/woopayments/settings-guide/#general).

To enable address autocomplete:

1.   Go to **WooCommerce > Settings > General**.
2.   Check the **Address autocomplete**box in the _General options_ section.
3.   Scroll to the bottom and click **Save Changes**. 

[![Image 22](https://woocommerce.com/wp-content/uploads/2024/08/Screenshot-taken-on-2025-10-22-at-18.59.41-UTC@2x.png?w=980)](https://woocommerce.com/wp-content/uploads/2024/08/Screenshot-taken-on-2025-10-22-at-18.59.41-UTC@2x.png)
This allows customers to more easily find and enter their entire address in the billing or shipping fields during checkout. It will work for any active payment method, so long as WooPayments is [enabled in checkout](https://woocommerce.com/document/woopayments/settings-guide/#general).

[![Image 23](https://woocommerce.com/wp-content/uploads/2024/08/Screenshot-taken-on-2025-10-22-at-19.03.49-UTC@2x.png?strip=all&w=704)](https://woocommerce.com/wp-content/uploads/2024/08/Screenshot-taken-on-2025-10-22-at-19.03.49-UTC@2x.png)
If more than one service is available for address autocompletion (e.g., if another plugin adds a different service), there will also be a **Preferred address autocomplete provider** setting shown. This allows you to pick the address search provider of your choice.

[![Image 24](https://woocommerce.com/wp-content/uploads/2024/08/Screenshot-taken-on-2025-10-22-at-20.09.26-UTC@2x.png?strip=all&w=704)](https://woocommerce.com/wp-content/uploads/2024/08/Screenshot-taken-on-2025-10-22-at-20.09.26-UTC@2x.png)
