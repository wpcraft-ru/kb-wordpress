# Testing WooPayments

This document provides instructions for testing payments, refunds, disputes, and the other functions of WooPayments. To learn more about WooPayments, including how to set it up on your store, see our [main documentation page](https://woocommerce.com/document/woopayments/).

**WARNING:** If you’re “testing” WooPayments by placing real orders while in live mode and then refunding yourself, the transaction fees associated with those orders [will **_not_** be refunded](https://woocommerce.com/document/woopayments/fees-and-debits/why-arent-fees-refunded/). Please use test mode for all test orders!

## [Enabling test mode](https://woocommerce.com/document/woopayments/testing-and-troubleshooting/testing/#enabling-test-mode)

[↑ Back to top](https://woocommerce.com/document/woopayments/testing-and-troubleshooting/testing/#doc-title "Back to top")
By default, WooPayments will accept real payment methods, such as cards, and actually charge your customers. To test payments, refunds, disputes, and other such processes without accepting real funds, you will need to enable test mode.

**NOTE:** Test mode does not charge any fees. Even if you still see the fees for an order listed in various places, this is only to show _what you would have been charged_ had the order been placed in live mode.

To enable test mode:

1.   Go to **Payments > Settings** in your site’s dashboard.
2.   Check the **Enable test mode** box.

[![Image 7](https://woocommerce.com/wp-content/uploads/2024/10/Screenshot-taken-on-2025-04-12-at-13.30.47-UTC@2x.png?strip=all&w=704)](https://woocommerce.com/wp-content/uploads/2024/10/Screenshot-taken-on-2025-04-12-at-13.30.47-UTC@2x.png)
1.   Select the **Enable** button in the confirmation popup box.
2.   Click **Save changes** at the bottom of the page.

## [Test orders](https://woocommerce.com/document/woopayments/testing-and-troubleshooting/testing/#test-orders)

[↑ Back to top](https://woocommerce.com/document/woopayments/testing-and-troubleshooting/testing/#doc-title "Back to top")
When test mode is enabled, any test payments processed via WooPayments will also create orders in WooCommerce. These orders include a notice on the edit order page stating that they were placed in test mode. This notice will be visible regardless of whether test mode is currently enabled or not.

[![Image 8](https://woocommerce.com/wp-content/uploads/2024/01/test-mode-banner.png?strip=all&w=704)](https://woocommerce.com/wp-content/uploads/2024/01/test-mode-banner.png)

A notice on the edit order page indicating that the order was placed in test mode.

If you need to delete all WooPayments test orders, you can do so under **WooCommerce > Status > Tools**. This will delete all orders placed via WooPayments while it was in test mode. It will _not_ delete any orders placed via other payment gateways. (Also, note that transaction data, such as that under **Payments > Transactions**, is not deleted.)

[![Image 9](https://woocommerce.com/wp-content/uploads/2025/11/Screenshot-taken-on-2025-11-04-at-14.27.33-UTC@2x.png?strip=all&w=704)](https://woocommerce.com/wp-content/uploads/2025/11/Screenshot-taken-on-2025-11-04-at-14.27.33-UTC@2x.png)

While test mode is enabled, the pages under the **Payments** menu item will only show test mode data. This includes data under **Payouts**, **Transactions**, and **Disputes**.

To remind store owners that test mode is currently enabled, a notice is also displayed at the top of any page under the **Payments** menu item.

## [Testing card payments](https://woocommerce.com/document/woopayments/testing-and-troubleshooting/testing/#testing-card-payments)

[↑ Back to top](https://woocommerce.com/document/woopayments/testing-and-troubleshooting/testing/#doc-title "Back to top")
To place a test order in test mode:

1.   Enable test mode as shown above.
2.   Go to your site’s shop and add a product to your cart.
3.   Proceed to the checkout page and fill out the form.
4.   Enter a [test card number](https://woocommerce.com/document/woopayments/testing-and-troubleshooting/testing/#test-cards), expiry date, and CVC code.
5.   Place your order.

The order will be processed just as if you had used a real card.

After you’re taken to the _Order received_ page, you can go back into your site’s dashboard and find the test transaction under **WooCommerce > Orders** and the **Payments > Transactions** page.

### [Manual capture](https://woocommerce.com/document/woopayments/testing-and-troubleshooting/testing/#manual-capture)

[↑ Back to top](https://woocommerce.com/document/woopayments/testing-and-troubleshooting/testing/#doc-title "Back to top")
To test a card payment authorization with manual capture:

1.   Go to **Payments > Settings** in your site’s dashboard.
2.   Enable test mode [as shown above](https://woocommerce.com/document/woopayments/testing-and-troubleshooting/testing/#enabling-test-mode).
3.   [Enable authorize and capture](https://woocommerce.com/document/woopayments/settings-guide/authorize-and-capture/#enabling-auth-and-capture).
4.   Place a test order as explained [here](https://woocommerce.com/document/woopayments/testing-and-troubleshooting/testing/#testing-card-payments).
5.   Go to **WooCommerce > Orders** and open the order you just placed.
6.   Confirm the order has a note in the sidebar stating: “A payment of $XX.XX was authorized using WooPayments.”
7.   [Capture the charge](https://woocommerce.com/document/woopayments/settings-guide/authorize-and-capture/#capturing-authorized-payments).
8.   Confirm the order now has a note in the sidebar like so: “A payment of $XX.XX was successfully captured using WooPayments.”
9.   Go to **Payments > Transactions.**
10.   Confirm that a corresponding transaction with the type _Charge_ is listed.

## [Testing disputes](https://woocommerce.com/document/woopayments/testing-and-troubleshooting/testing/#disputes)

[↑ Back to top](https://woocommerce.com/document/woopayments/testing-and-troubleshooting/testing/#doc-title "Back to top")
To simulate [a dispute](https://woocommerce.com/document/woopayments/fraud-and-disputes/managing-disputes/) with WooPayments, place a test order as shown above using one of the cards from this table:

| Description | Number | Details |
| --- | --- | --- |
| Transaction not authorized | 4000 0000 0000 0259 | The charge will be disputed as unauthorized. This type of dispute is protected after 3D Secure authentication. |
| Product not received | 4000 0000 0000 2685 | With default account settings, charge succeeds, only to be disputed as product not received. This type of dispute isn’t protected after 3D Secure authentication. |

The order will be processed, but the payment will be disputed immediately. You can then view and respond to the dispute by going to **Payments > Disputes** in your site’s dashboard.

### [Dispute responses](https://woocommerce.com/document/woopayments/testing-and-troubleshooting/testing/#dispute-responses)

[↑ Back to top](https://woocommerce.com/document/woopayments/testing-and-troubleshooting/testing/#doc-title "Back to top")
After creating a dispute with a dispute test card above, you can also test the dispute response process.

1.   Go to **Payments > Disputes** in your site’s dashboard.
2.   Click the dispute in the list to open it up.
3.   Click **Challenge Dispute**.
4.   Click **Next** for each section (or click the Review tab in the top navigation) until you see the _Cover Letter_ field.
5.   In the _\_Cover Letter\__ field, remove all text and do one of the following: 
    *   Enter `winning_evidence` to win the dispute.
    *   Enter `losing_evidence` to lose the dispute.

6.   At the bottom, click **Submit**.

You can also accept the dispute in Step 3 instead of challenging it to test that scenario.
