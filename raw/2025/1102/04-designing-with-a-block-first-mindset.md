## Transcript

### Introduction
Designing with a block-first mindset. We will explore block composition for WordPress block themes. By the end, you'll understand how to approach design with a block-first mentality, creating modular and reusable components for your website.

### Modularity
In WordPress block themes, we shift from traditional page layouts to a modular approach. Block-first design means breaking down your website into smaller, reusable components that can be mixed and matched. Think of blocks as LEGO pieces — you can create intricate website designs using fundamental blocks. This streamlines your design process and enhances consistency.

### Blocks
Blocks allow you to create complex designs without writing code. Blocks connect to `theme.json` which sets default styles. In the Site Editor, Global Styles lets you tweak defaults using visual controls, ensuring design consistency while maintaining flexibility.

Blocks adhere to open web standards — portable across platforms. The structured nature of blocks makes them compatible with AI tools, opening possibilities for automated design suggestions.

### Reusable components
Typical designs to create as patterns:

1. **Featured item** — product highlight, key blog post, or pricing table. Design once, use throughout.
2. **Newsletter sign-up** — visually appealing form, easily placed on various pages.
3. **Testimonial** — customer quotes block, used individually or grouped.
4. **Call-to-action** — eye-catching CTAs.

### Block composition principles

1. **Start small** — begin with essentials in your Style Book: text, images, buttons. These form the foundation.
2. **Build up** — combine basic elements into more complex blocks. Create a media and text pattern, save as non-synced pattern, design the layout.
3. **Think responsively** — design blocks to work on all screen sizes. Use Columns block with "stack on mobile". Preview on tablet and mobile.

**Pattern anatomy example:**
- Group block (container) → Columns block
- Left column: Paragraph, Heading, Paragraph, Buttons (CTA)
- Right column: Image block

### Conclusion
When you adopt a block-first mindset, you're designing a flexible system of components, not just pages. Think modularly, design for reusability using blocks and patterns.
