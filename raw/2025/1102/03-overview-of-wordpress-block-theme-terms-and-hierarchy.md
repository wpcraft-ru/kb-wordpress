### Architecture: Pages versus posts
In block themes, everything is built with blocks, but WordPress still uses two main types of content: pages and posts.

1. **Pages** — static, timeless content that remains consistent over time. Made solely of blocks, you may easily construct different layouts for each page as needed.
2. **Posts** — dynamic content typically used for blogs or news updates. Time-based entries displayed in reverse chronological order. Fully customizable with blocks, organized using categories and tags.

### Block theme hierarchy (smallest to largest)

#### 1. Blocks
The foundational element. Individual content elements like paragraphs, images, buttons, or parent containers like columns, groups and galleries.

#### 2. Patterns
Pre-designed arrangements of blocks that can be easily inserted and customized. Reusable designs you can add to pages or posts.

#### 3. Template Parts
Editable, reusable sections of a website primarily used for structural elements with specific semantic meanings. Typically represent headers, footers, and sidebars. Built with blocks and customizable in the Site Editor.

#### 4. Templates
Define the overall structure of different page and post types. Fully customizable using blocks and incorporate template parts. Determine layout for specific content types like pages, single posts, archives, etc.

#### 5. Theme
The overarching design framework for your entire WordPress site. Defines overall look, feel, and behavior. Includes all templates, template parts, block patterns, and Styles.

### Flexibility
Block themes' hierarchy offers flexibility and makes customizing design easier. You can create and customize at any level through the Site Editor, from individual blocks to entire themes.

**Design workflow example:**
1. Design basic content elements (text styles, buttons, image layouts)
2. Create patterns by combining blocks into reusable layouts (testimonials, CTAs, product showcases)
3. Develop header and footer template parts reflecting brand
4. Build out full templates

### Conclusion
Understanding WordPress block themes hierarchy helps you plan and implement your designs. You'll see how each component fits into the larger picture and how to use the Site Editor to customize your theme.
